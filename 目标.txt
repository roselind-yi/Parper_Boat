纸船漂流 —— 情感交互叙事游戏
项目背景
面向年轻用户的情感叙事类互动体验产品。用户选择收信人、在纸船上书写私密内容、折纸后投入虚拟水域，纸船经历六章漂流旅程最终抵达归宿。需构建一套融合实时物理渲染、多人联机交互、AI 剧情生成的完整游戏系统。

具体工作内容
1. 后端服务架构设计

使用 Spring Boot 搭建游戏后端服务，设计用户系统（JWT 认证 + BCrypt 加密）、漂流瓶系统、好友系统三大模块
数据库采用 MySQL 8.0 存储用户关系、漂流记录、信件内容等结构化数据，Redis 缓存在线用户状态与热门漂流瓶
设计 RESTful API 接口（用户注册/登录、投递纸船、打捞纸船、查看漂流轨迹、好友互动），Swagger 自动生成接口文档
2. 前端渲染引擎架构

设计多层 Canvas（2-9 层动态叠加）渲染管线：底层 Three.js WebGL 处理水面物理模拟（波纹传播、反射计算、光照模型），中层 Canvas 2D 处理纸船实体与粒子特效，顶层处理 UI 与文字叠层
实现 Canvas 池化复用机制，预分配 Canvas 对象池避免频繁创建销毁导致 GC 卡顿，渲染帧率从 35fps 提升至稳定 60fps
章节状态机模型：定义六章旅程的状态节点与转移条件，通过事件驱动状态流转，支持中途存档、回退与断点续玩，存档数据 JSON 序列化后持久化至 MySQL
3. WebSocket 实时联机系统

基于 Spring WebSocket + STOMP 协议构建实时通信层，实现多人同时在线漂流——用户投递纸船后广播至全水域，其他用户可实时打捞、回复
设计心跳检测机制与断线重连策略，连接断开后自动恢复会话状态
在线用户列表通过 Redis Set 维护，支持按水域分区广播
4. 漂流瓶匹配算法

基于用户兴趣标签（情感倾向/年龄/地域）设计协同过滤匹配算法，纸船投递后优先推送给匹配度高的用户
引入随机因子避免匹配结果固化，保证低活跃用户也有曝光机会
5. AI 剧情生成引擎（规划中）

集成大语言模型 API（GPT/文心），根据用户书写的信件内容自动生成六章漂流旅程中的随机事件与 NPC 对话
设计 Prompt 模板：系统角色设定 → 当前章节背景 → 用户信件内容注入 → 生成下一段剧情，通过 LangChain 管理 Prompt 链式调用
基于 NLP 情感分析模型对用户信件内容打分，动态调整漂流行进速度、天气效果（晴天/阴天/暴风雨）与结局走向
6. 工程化与运维

Docker Compose 一键部署（Spring Boot + MySQL + Redis + Nginx），Nginx 反向代理 + 静态资源分离
GitHub Actions 配置 CI/CD 流水线，代码推送后自动构建镜像并部署至阿里云 ECS
版本迭代管理：经历 v1.4.0 → v2.3.0 共 7 个迭代版本，Git Flow 分支管理（feature/develop/main）
项目成果
Canvas 池化优化使渲染帧率从 35fps 提升至稳定 60fps，六章完整叙事旅程
WebSocket 实时联机支持多人同时漂流交互，Redis 维护在线用户状态
7 次版本迭代从单场景 Demo 演进为完整情感叙事游戏
